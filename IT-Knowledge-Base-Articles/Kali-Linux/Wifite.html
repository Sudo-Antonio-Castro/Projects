    [KB] Wifite

    Audience: Help Desk / IT Admin System: Linux / Browser Last reviewed: 2025-12-17

    This guide supports an authorized wireless security lab simulation using Kali Linux to validate that a lab SSID can be detected, a WPA/WPA2 validation artifact (e.g., handshake evidence) can be captured, and an offline dictionary-based password strength audit can be completed using an approved wordlist. Outcomes help confirm wireless testing readiness and highlight risks associated with weak passphrases in a controlled environment.

    Good for
    Authorized wireless security assessments in a lab or approved engagement, validating exposure and producing remediation focused findings (e.g., hardening recommendations and documented results).
    Be careful
    Use only Approved Networks and only with Explicit written authorization and a defined scope/time window. Do NOT test public networks or anything outside scope, and stop immediately if you see unintended impact.

    Purpose

    To provide a repeatable, step-by-step process for conducting a controlled Wi-Fi security assessment in a lab environment using Kali Linux and Wifite. This procedure demonstrates handshake capture and dictionary-based password testing to help teams identify weak credentials, understand wireless attack paths, and improve security hardening—only on networks explicitly owned or authorized for testing.

    When to use this guide

    • When conducting training for wireless security concepts (handshake capture + offline cracking) in a controlled environment.
    • When validating a wireless test network configuration (e.g., confirming WPA2 is enabled and that credentials meet complexity expectations).
    • When demonstrating the impact of weak passphrases using approved wordlists for security awareness.
    • When reproducing a known lab scenario for documentation, troubleshooting, or repeatability.
    Security note (recommended)

    Only work on networks explicitly owned or authorized for testing.

    Procedure

    1. Step 1 — Action: Prepare the dictionary file for Wifite, do so by locating the dictionary file.
      2. locate rockyou
    2. Step 2 — Action: Now decompress the file.
      3. sudo gunzip /usr/share/wordlists/rockyou.txt.gz
    3. Step 3 — Action: Run wifite with the following arguments.
      4. sudo wifite --dict /usr/share/wordlists/rockyou.txt
    4. Step 4 — Action: Once wifite starts scanning for available networks, press CTRL + C to stop the scan, then select the target network number.
    5. Step 5 — Info: Once the handshake is captured, wifite will automatically start cracking the handshake.
    6. Step 6 — Info: Once the handshake is cracked, it will automatically show the password on the screen.
