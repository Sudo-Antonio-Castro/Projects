Client-Side Virtualization

Virtualization lets one physical computer run multiple isolated operating systems. It is useful for labs, testing, compatibility, training, and sandboxing.

Hypervisors and VMs

  • A hypervisor creates and manages virtual machines that share the host hardware.
  • Type 1 hypervisors run closer to the hardware, while Type 2 hypervisors run on top of a host operating system.
  • Each VM needs virtual CPU, memory, storage, and network resources assigned to it.

Host Requirements

  • Virtualization works best on systems with sufficient CPU support, hardware virtualization extensions, memory, and storage performance.
  • Running several guests at once can consume RAM and disk capacity very quickly.
  • Snapshots let you capture a state and roll back changes, but they also increase storage usage.

Security and Use Cases

  • Virtual machines help isolate workloads, test software safely, and simulate networks or server roles.
  • Guest operating systems still require patching and malware protection.
  • Unmanaged or unauthorized virtual machines can create support and security problems just like untracked physical systems.
Summary

Virtualization is about isolation and flexibility, not free performance. Every guest still needs real host resources behind it.

A+ Exam Tip

If an exam scenario mentions testing, sandboxing, or running another OS without changing the main computer, think virtualization first.

Cloud Concepts

Cloud computing delivers infrastructure, platforms, desktops, and software over a provider-managed environment. The main exam task is telling the models apart.

Service Models

  • Infrastructure as a Service (IaaS) provides virtual servers, storage, and networking that the customer configures.
  • Platform as a Service (PaaS) provides the application platform and development environment, while Software as a Service (SaaS) delivers a finished application to end users.
  • Desktop as a Service (DaaS) streams a managed desktop environment from the cloud.

Deployment Models

  • Public cloud is provider-owned and shared, private cloud is dedicated to one organization, and hybrid cloud mixes local and provider resources.
  • Community cloud describes shared infrastructure for organizations with similar needs.
  • Elasticity, measured service, and rapid provisioning are key cloud characteristics.

Storage and Operations

  • Cloud file storage services sync data across devices and are common in both personal and business workflows.
  • Availability and responsibility are shared - providers manage the platform, but customers still manage their data, accounts, and configurations.
  • Technicians should be able to explain where control shifts between customer and provider.
Summary

Cloud models differ by how much the provider manages for you. Once you know who owns the application, platform, and infrastructure layers, the right label usually becomes clear.

A+ Exam Tip

IaaS gives you building blocks, PaaS gives you a development platform, SaaS gives you the finished app, and DaaS gives you the desktop itself.
