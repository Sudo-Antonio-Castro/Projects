Networking, Routing and Firewall Rules

Interfaces, addressing, routes, name resolution, and packet filtering.

Interface management

  • The ip tool family is central to modern Linux interface and route work.
  • Interfaces need correct addressing, state, and persistence.
  • NetworkManager tooling may be used on many modern distributions.

Routing and resolution

  • Default routes send traffic to non-local networks.
  • Static routes can direct traffic through specific gateways and devices.
  • DNS maps hostnames to IP addresses and is foundational for service access.

Firewall controls

  • Host firewalls enforce traffic policy.
  • Administrators must understand service-based rules, zones, and direct rule behavior.
  • Packet redirection and NAT rules should be applied deliberately and tested carefully.
Quick takeaway

Use this module to connect commands to operational intent. Linux+ rewards knowing which tool to reach for first and why.
