Package Management and Software Sources

Rpm and deb ecosystems, repositories, dependencies, and libraries.

Package ecosystems

  • RPM-based and DEB-based distributions use different packaging tools.
  • Tool choice usually follows the distribution family rather than personal preference.
  • Administrators must understand both local package installation and repository-based workflows.

Dependencies and repositories

  • Repository-aware tools resolve dependencies and metadata automatically.
  • Local and remote repositories support repeatable software delivery.
  • Repository governance helps standardize builds across multiple systems.

Libraries and change control

  • Shared libraries reduce duplication between applications.
  • Broken dependencies can prevent software from starting.
  • Before removing software, confirm what depends on it and how the change affects production.
Quick takeaway

Use this module to connect commands to operational intent. Linux+ rewards knowing which tool to reach for first and why.
