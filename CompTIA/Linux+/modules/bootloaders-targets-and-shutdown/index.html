Bootloaders, Targets and Shutdown

Grub2, boot troubleshooting, systemd ordering, and controlled shutdown.

Boot sequence

  • Firmware starts the machine, then the bootloader loads the kernel and initramfs.
  • The kernel initializes hardware and hands off to PID 1.
  • Knowing the sequence helps you isolate failures quickly.

GRUB2 and recovery

  • GRUB2 selects kernels and passes startup parameters.
  • Installed kernel packages and the running kernel version help verify which boot image belongs in /boot.
  • Recovery options are often managed from the bootloader before userspace is available.

systemd startup

  • Targets replace old runlevel thinking on modern distributions.
  • Unit ordering uses directives such as After= and Before=.
  • Graceful reboot and shutdown protect service state and reduce data loss risk.
