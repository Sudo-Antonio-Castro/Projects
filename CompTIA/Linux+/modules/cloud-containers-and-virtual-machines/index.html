Cloud, Containers and Virtual Machines

Cloud models, containers, snapshots, images, and virtualization basics.

Virtualization basics

  • Hypervisors manage and run virtual machines.
  • Guests include their own operating system, unlike containers.
  • Network mode selection affects how virtual systems reach the rest of the network.

Containers

  • Containers share the host kernel and generally use less overhead than full VMs.
  • Container logs and lifecycle commands are part of routine operations.
  • Image management and runtime inspection matter for support work.

Cloud thinking

  • Cloud services deliver on-demand resources rather than fixed local-only capacity.
  • Linux roles often include cloud-hosted services and automation.
  • Operational consistency matters across virtual and cloud-hosted systems.
