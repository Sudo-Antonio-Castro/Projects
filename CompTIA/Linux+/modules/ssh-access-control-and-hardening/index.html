SSH, Access Control and Hardening

Sudo, mac controls, keys, tunnels, and secure service exposure.

Privilege and remote access

  • sudo supports least privilege and auditing better than full-time root use.
  • SSH is central to secure remote administration on Linux.
  • Key-based authentication can be safer and more convenient than passwords alone.

Hardening controls

  • SELinux and AppArmor provide additional confinement controls on many systems.
  • Unused services and accounts should be removed or disabled.
  • Secure defaults reduce attack surface before incidents happen.

Secure service exposure

  • SSHD configuration controls remote access behavior such as port settings and auth options.
  • SSH tunneling forwards traffic securely through an encrypted session.
  • Hardening is ongoing, not a one-time event.
