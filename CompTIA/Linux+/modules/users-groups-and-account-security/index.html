Users, Groups and Account Security

Identity records, group design, password policy, and least privilege.

Identity records

  • /etc/passwd stores basic account data and /etc/shadow stores password hashes and aging information.
  • Groups provide a scalable way to manage shared access.
  • Primary and supplementary groups support flexible privilege design.

Account administration

  • User lifecycle tasks include creation, locking, unlocking, shell management, expiry control, and removal.
  • Password aging and account locking improve control over access.
  • Least privilege should guide membership and shell access.

Troubleshooting access

  • Bad shell settings, missing home directory permissions, and group errors can all break login or application behavior.
  • Directory execute permissions affect path traversal.
  • Identity problems often show up as file access or service issues.
