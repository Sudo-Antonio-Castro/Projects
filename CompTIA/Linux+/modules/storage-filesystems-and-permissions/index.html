Storage, Filesystems and Permissions

Partitioning, lvm, mount persistence, acls, and ownership.

Storage layout

  • Linux storage combines partitions, logical volumes, filesystems, and mount points.
  • GPT supports larger disks and more flexible layouts than classic MBR.
  • LVM adds flexibility for resizing and storage lifecycle management.

Filesystems and mounts

  • Persistent mounts are commonly defined in /etc/fstab.
  • ext4 and XFS remain common Linux filesystem choices.
  • File system checks, formatting, and mount troubleshooting are routine admin tasks.

Permissions and ACLs

  • Traditional rwx permissions work with ownership and groups.
  • ACLs allow more granular access when classic permission bits are not enough.
  • umask shapes the default permission bits removed from newly created files and directories.
