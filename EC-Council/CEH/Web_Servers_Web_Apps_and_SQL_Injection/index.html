Lesson 1: Web Targets

Servers and applications under attack

Web infrastructure exposes public-facing services that often process high-value information.

Focus points

  • Misconfigurations, outdated content, weak certificates, and verbose error handling create risk.
  • Web application flaws may involve cookies, input validation, session handling, or hidden fields.
  • Attackers may mirror sites, inspect source, or enumerate HTTP behaviors before exploitation.

Summary

Because web systems are public by design, small mistakes can become large exposures very quickly.

Exam angle

If a question references user input, sessions, cookies, or page behavior, suspect a web application issue.

Lesson 2: SQL Injection

Abusing the data layer

SQL injection occurs when untrusted input changes the meaning of a database query.

Focus points

  • Poor input handling can let an attacker read, modify, or manipulate backend data.
  • Attack types vary, but the core problem is unsafe query construction.
  • Countermeasures include input validation, parameterized queries, and careful server-side handling.

Summary

SQL injection is ultimately about controlling the query logic rather than just sending bad characters.

Exam angle

Look for answers that keep user input separate from SQL command structure.

Module focus

Server exposure, web flaws, input abuse, and injection-driven compromise.

Why it matters

Web targets stay popular because they are visible, complex, and business-critical.
